Machine Control Side

Procedure 1: Column PC Setup

Configures the primary host machine, Siemens Operate, and SProxy control networks.

Prerequisite: Download Required Files

The Technician needs to go to the Applications Teams Channel and download the whole Tarus Tech folder. This folder has the latest TarusNC version, the latest Default Configuration File for an LCMT 5 axis Baserail, the Software Password Folder for daily passwords, the desktop images used for the two PCs, among many other very useful files and applications.

Critical Warning

Using a default machine configuration on a live machine can cause catastrophic damage. Ensure you have the specific job ConfigurationFiles (e.g., 3529-Column) downloaded before finalizing the software setup.

1

Initial Windows Configuration

  1. Power on the PC. Select United States for the region and US for the keyboard layout. Click Skip for the second layout.
  2. When prompted to connect to a network, select I don't have Internet, then click Continue with limited setup. Accept the Windows License Agreement.
  3. Create a local account using Username: Tarus and Password: Tarus123. (Select the security questions in order and use Tarus as the answer for all 3).
  4. On the Privacy Settings screen, toggle ALL options to OFF and click Accept.
  5. Once at the Windows desktop, connect to the tarusguest Wi-Fi network to gain internet access.
  6. Go to Start > Settings > System > About. Click Rename this PC and name it [JobNum]-Column. Select Restart later.
  7. Right-click the Start Menu and select Run. Type UAC and press Enter. Drag the slider all the way down to Never notify and click OK.
  8. Go to Start > Settings > System > Notifications & actions. Toggle the main "Get notifications from apps and other senders" to Off.
  9. Go to Settings > Time & Language > Date & time. Change the Time Zone to Eastern Time. Toggle Set time automatically to On, then click Sync now.
  10. Go to Settings > Update & Security. Click Check for updates. Install all available updates. Restart and repeat until the page says "You're up to date".
2

Prep, BIOS & System Hardening

  1. System Permissions: Open File Explorer. Right-click the C: drive and select Properties > Security tab > Edit. Check the Allow box for Full Control for ALL listed Users and Groups. Click Apply and OK.
  2. Open View network connections from the Start menu. Identify the port for the private machine network and rename it to Machine Control. Identify the port for the remote desktop and rename it to Customer Network.
  3. Restart the PC and press Esc to enter the BIOS. In the Advanced tab, set "After Power Loss" to Power On. Save and Exit.
  4. Right-click the Start button and select Windows PowerShell (Admin).
  5. Type irm "https://christitus.com/win" | iex and press Enter.
  6. Navigate to the Tweaks tab and click the Standard profile button. Ensure the following additional advanced tweaks are explicitly checked:
    • Create Restore Point
    • Edge Debloat
    • Remove Xbox & Gaming Components
    • Set Classic Right-Click Menu
    • Set Display for Performance
  7. Under the Performance Plans section on the right side of the utility, click Add and Activate Ultimate Performance Profile.
  8. Leave the WinUtil window open. Click back into the blue Admin PowerShell window running behind it, paste the following command, and hit Enter. This foolproof script creates the plan, brands it, activates it, and enforces strict CNC power timeouts: 
    $p = powercfg /list | Select-String "TARUS - Ultimate"; if (-not $p) { $n = powercfg -duplicatescheme e9a42b02-d5df-448d-aa00-03f14749eb61; $guid = ($n -split '\s+')[3]; powercfg /changename $guid "TARUS - Ultimate Power Plan" "Ultimate Performance for Tarus Control" } else { $guid = ($p.ToString() -split '\s+')[3] }; powercfg /setactive $guid; powercfg /change standby-timeout-ac 0 2>$null; powercfg /change monitor-timeout-ac 0 2>$null; powercfg /change hibernate-timeout-ac 0 2>$null; powercfg /change disk-timeout-ac 0 2>$null; powercfg /setacvalueindex $guid 2a737441-1930-4402-8d77-b2bea128a440 48e6b7a6-50f5-4782-a5d4-53bb8f07e226 0 2>$null; powercfg /setacvalueindex $guid 501a4d13-42af-4429-9fd1-a8218c268e20 ee12f906-d277-404b-b6da-e5fa1a55844a 0 2>$null; powercfg /setactive $guid; powercfg /list | Select-String "Power Scheme GUID" | Where-Object { $_ -notmatch $guid -and $_ -notmatch "8c5e7fda-e8bf-4a96-9a85-a6e23a8c635c" } | ForEach-Object { powercfg /delete ($_ -split '\s+')[3] 2>$null }; Write-Host "TARUS Power Plan Created, Activated & Enforced." -ForegroundColor Yellow
  9. Click Run Tweaks at the bottom of the utility and wait for it to complete.
  10. Navigate to the Updates tab and select Security Settings. Close the utility.
  11. Open the Windows Control Panel > Power Options. Verify the newly active plan is TARUS - Ultimate Power Plan. Click Change plan settings. Click Change advanced power settings. Expand Hard disk and verify "Turn off hard disk after" is set to 0. Click Apply and OK.
3

Core Software & Configuration Files

  1. Create a C:\Temp folder. Unzip the Siemens installer here. Run SINUMERIK_OPERATE_pc.exe as Administrator. (Select Standard ONE NCU and Complete setup). Restart when prompted.
  2. Open the Windows Control Panel and set View by to Small icons. Look for Communication Settings.

    💡 Missing Icon? It is common for the applet to fail to register on the first install. If "Communication Settings" is missing, run SINUMERIK_OPERATE_pc.exe again, select Repair, and restart. Repeat this Repair/Restart loop as many times as needed until the icon successfully appears.

    Once open, select Access points. Expand S7ONLINE and select the Machine Control adapter.
  3. Run the TarusNC installer as Administrator. (Select TarusNC Only and SProxy). Restart the PC.
  4. Install Utilities: Install Google Chrome and Notepad++ from your downloaded Tarus Tech folder. Go to Settings > Apps > Default apps. Set the Web browser to Google Chrome, and set .txt files to open with Notepad++.
  5. Hide the Siemens Operate UI: Open File Explorer and navigate to exactly: C:\Program Files (x86)\Siemens\MotionControl\siemens\sinumerik\hmi\cfg.
    • Step A: From your downloaded Tarus Tech folder, copy the pre-configured systemconfiguration - hide.ini file and paste it directly into this directory.
    • Step B: Rename the original systemconfiguration.ini to systemconfiguration - Show.ini.
    • Step C: Rename the pasted systemconfiguration - hide.ini to just systemconfiguration.ini.

    💡 How to un-hide later: If you need to see the Siemens window for diagnostics, open Task Manager and end TarusNC, SProxy, and all Siemens tasks. Rename the active systemconfiguration.ini to systemconfiguration - Hide.ini, and rename the systemconfiguration - Show.ini back to systemconfiguration.ini. You should only ever have two configuration files in this folder at a time.

    (Note: If doing this manually without the downloaded file, open Notepad++ as Administrator, duplicate the original file, edit the active one, locate the [dialogs] section, and comment out every DLG=... line AND the Startup Area=AreaMachine line by adding a ; to the beginning of each line.)
4

Network IPs, Graphics & Handoff

  1. Right-click the Machine Control adapter and select Properties. Uncheck everything EXCEPT Internet Protocol Version 4 (TCP/IPv4). Click IPv4 > Properties and set the IP address to 192.168.214.55 with Subnet Mask 255.255.255.0. Leave Gateway blank. Return to the main Properties window, click Configure. Go to the Power Management tab and uncheck BOTH power saving boxes. Go to the Advanced tab and set Energy-Efficient Ethernet to Off.
  2. Navigate to C:\Tarus Products\. Delete the default ConfigurationFiles folder and paste in the specific folder for this job from your downloaded files.
  3. Right-click the Customer Network adapter and select Properties. Uncheck all items EXCEPT Client for Microsoft Networks, File and Printer Sharing, and Internet Protocol Version 4 (TCP/IPv4). Select IPv4 > Properties, and ensure it is set to Obtain an IP address automatically (DHCP). Return to main Properties, click Configure > Power Management and uncheck Allow the computer to turn off this device. Set Energy-Efficient Ethernet to Off.

    Crucial Wi-Fi Step: Repeat this power-saving disable process for the Wi-Fi adapter. Right-click your Wi-Fi adapter, click Properties > Configure > Power Management, and ensure Allow the computer to turn off this device is UNCHECKED.
  4. Configure Windows Graphics: Go to Start > Settings > System > Display > Graphics settings. Click Browse and locate TarusNC.exe. Click the application in the list, select Options, and choose High performance to force Windows to use the NVIDIA GPU.
  5. Configure NVIDIA Control Panel: Open the NVIDIA Control Panel > Manage 3D Settings. Set the Global Power Management Mode to Prefer Maximum Performance. Under Program Settings, add TarusNC.exe and set it to High Performance. Finally, navigate to the Configure Surround, PhysX tab and set the Processor directly to your NVIDIA GPU.
  6. Go to Settings > System > Remote Desktop and toggle it to On. Click Confirm.
  7. From your downloaded Tarus Tech folder, copy the TARUS Column PC logo.png file into the Pictures folder. Right-click the Desktop > Personalize, and set it as the Desktop Background.
  8. Create IT Handoff Documentation: Right-click the Start Menu, select Run, type cmd and hit Enter. In the black window, type ipconfig /all and press Enter. Record the Physical Address (MAC) for the Customer Network adapter. Provide the customer IT department with the following info:
    • PC Name: [Job Number]-Column
    • MAC Address: [Physical Address]
    • Required By: [Job Number]-Desk
    • Purpose: Requires RDP access through firewall policies.

Remote Desktop Configuration & Test

  1. Open the Remote Desktop Connection app (Click the Start Menu, type Remote Desktop Connection, and open it).
  2. Configure the connection: Enter the Computer name [JobNum]-Column.
    CAUTION: You must use the PC Name, not an IP address. This ensures it will work on the customer's network once their IT department has configured it.
  3. Click the Show Options button. Enter Username: Tarus and check the box for Allow me to save credentials.
  4. CRITICAL - Security Prompt Fix: Go to the Local Resources tab. Uncheck BOTH Printers and Clipboard. Next, click the More... button at the bottom. You MUST uncheck every single box in this list (Smart cards, WebAuthn, Ports, Drives, etc). Click OK.
    💡 By completely disabling local resource sharing, Windows is forced to bring back the missing "Don't ask me again" checkbox during the security prompt later in this step!
  5. Go to the Display tab and check Use all my monitors for the remote session.
  6. Go back to the General tab, click Save As..., and save the file to the Desktop as Connect to Machine Control. Close the Remote Desktop app.
  7. Manual Connection Test: On the desktop, double-click the new `Connect to Machine Control` icon. Enter the password Tarus123 and critically, check the box for Remember me to save the password into Windows.
  8. Click OK to connect. Because you disabled resources in step 4, the yellow security prompt will now have a visible Don't ask me again for connections to this computer checkbox. Check this box and click Yes. The prompt will never appear again.
  9. Confirm the connection is successful. You should see the Column PC's desktop. Once confirmed, disconnect by clicking the "X" on the blue RDP bar at the top of the screen.
4

Kiosk Automation & Final Polish

  1. Force Auto-Login Checkbox (Registry Fix): Right-click the Start Menu and select Run. Type regedit and press Enter (Click Yes to UAC). In the address bar at the top, paste this exact path:
    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\PasswordLess\Device On the right-hand side, double-click DevicePasswordLessBuildVersion and change the "Value data" to 0. Click OK and close the Registry Editor.
  2. Configure Auto-Login: Right-click the Start Menu again and select Run. Type netplwiz and press Enter.
  3. Select the Tarus user. Uncheck Users must enter a username and password to use this computer and click Apply. An "Automatically sign in" window will pop up. Enter Password: Tarus123 (confirm the password), then click OK to close both windows.
  4. Create Prompt-Free Startup Shortcut: Right-click an empty space on the Desk PC desktop, select New > Shortcut. In the location box, paste this exact command (Make sure to replace [JobNum] with your actual Job Number!): 
    cmd /c start /high mstsc.exe /v:[JobNum]-Column /multimon /f
    Click Next, name the shortcut Machine Control, and click Finish.
    💡 Security Prompt Fix: By launching the connection directly via this command-line parameter instead of using an unsigned .rdp file, Windows completely skips the new un-hideable "Unknown remote connection" prompt!
  5. Set RDP to Launch on Startup: Right-click the Start Menu and select Run. Type shell:startup and hit Enter. Drag (or Cut/Paste) the newly created Machine Control shortcut into this Startup folder.
    Leave the original `.rdp` file physically on the desktop so technicians can easily double-click it for maintenance in the future.
  6. From your downloaded Tarus Tech folder, copy the TARUS Desk PC Status Message logo.png file into the Pictures folder and set it as the Desktop Background.
  7. Right-click the Desktop, select View, and uncheck Show desktop icons for a clean kiosk appearance. Restart the PC to verify it logs in and connects automatically.
Support & Diagnostics

Troubleshooting & Maintenance

Resolves common network drops and handles credential/history management.

Network Stack Corruption

If Remote Desktop fails despite verifying Ethernet cables and pinging the IP successfully, the Windows networking framework may be corrupt.

  1. Launch WinUtil (Right-click Start Menu > Windows PowerShell (Admin) > type irm christitus.com/win | iex)
  2. Navigate to the Config Tab.
  3. Click Reset Network under Fixes.
  4. Restart PC and re-apply static IPv4 settings.

High Latency / Video Lag

If the Desk PC connects but the interface is severely lagging or dropping frames, check the Column PC graphics settings.

  1. Go to Settings > System > Display > Graphics settings on the Column PC.
  2. Ensure Hardware-accelerated GPU scheduling is OFF.
  3. Ensure TarusNC.exe is set to High Performance.

Clear RDP Connection History

How to completely wipe saved passwords and clear old computer names out of the Remote Desktop dropdown menu.

  1. Step A: Remove Saved Passwords
  2. Click Start, type Credential Manager, and hit Enter.
  3. Click on Windows Credentials and scroll to Generic Credentials.
  4. Look for entries starting with TERMSRV/. Click the down arrow and select Remove.
  5. Step B: Clear Dropdown History (Registry)
  6. Right-click Start Menu, select Run, type regedit and hit Enter.
  7. Navigate to:
    HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Default
  8. In the right pane, right-click any MRU entries (e.g., MRU0, MRU1) you want to remove and select Delete. Close the Registry Editor.

Revoke RDP User Access

How to permanently revoke a user's permission to remotely access the Column PC.

  1. Click Start, type Advanced system settings, and hit Enter.
  2. Go to the Remote tab at the top.
  3. Under the "Remote Desktop" section, click the Select Users... button.
  4. Select the user you want to revoke access from and click Remove.
  5. Click OK, and then OK again to apply the changes.
Note: Administrators automatically have access. To block all connections entirely, select the "Don't allow remote connections to this computer" option on the Remote tab.
Field Operations

Technician Toolkit

1-click PowerShell commands to bypass customer IT policies, verify network drops, and audit hardware natively.

Warning: Administrative Privileges Required

These tools modify core system behaviors. You must right-click the Start Menu and run Windows PowerShell (Admin) before pasting these scripts.

1. Hardware & Network Audit (WMI)

Safe to Run Anytime

Instantly pulls Motherboard, CPU, GPU, RAM, and active MAC Addresses directly from the BIOS. Bypasses the need to download 3rd-party tools like Speccy.

Write-Host "--- TARUS SYSTEM AUDIT ---" -ForegroundColor Cyan; $sys = Get-CimInstance Win32_ComputerSystem; Write-Host "System: $($sys.Manufacturer) $($sys.Model)"; $cpu = Get-CimInstance Win32_Processor | Select-Object -First 1; Write-Host "CPU:    $($cpu.Name)"; $gpu = Get-CimInstance Win32_VideoController | Select-Object -First 1; Write-Host "GPU:    $($gpu.Name)"; $ram = Get-CimInstance Win32_PhysicalMemory | Measure-Object -Property Capacity -Sum; Write-Host "RAM:    $([Math]::Round($ram.Sum / 1GB)) GB"; Write-Host "--- ACTIVE MAC ADDRESSES ---" -ForegroundColor Cyan; Get-NetAdapter | Where-Object Status -eq "Up" | Select-Object InterfaceAlias, MacAddress | Format-Table -AutoSize

2. CNC Ultimate Power Enforcer

Permanent Fix

CNC controls require uninterrupted heartbeats. If customer IT policies enforce sleep settings, the SProxy/Siemens communication will drop out in micro-seconds, crashing the machine.

What this does: Intercepts the active power plan via registry (PowerCfg) and forces Display, Disk, Sleep, Hibernate, USB Selective Suspend, and PCIe Link State Power Management to 0 (Disabled).

$p = powercfg /list | Select-String "TARUS - Ultimate"; if (-not $p) { $n = powercfg -duplicatescheme e9a42b02-d5df-448d-aa00-03f14749eb61; $guid = ($n -split '\s+')[3]; powercfg /changename $guid "TARUS - Ultimate Power Plan" "Ultimate Performance for Tarus Control" } else { $guid = ($p.ToString() -split '\s+')[3] }; powercfg /setactive $guid; powercfg /change standby-timeout-ac 0 2>$null; powercfg /change monitor-timeout-ac 0 2>$null; powercfg /change hibernate-timeout-ac 0 2>$null; powercfg /change disk-timeout-ac 0 2>$null; powercfg /setacvalueindex $guid 2a737441-1930-4402-8d77-b2bea128a440 48e6b7a6-50f5-4782-a5d4-53bb8f07e226 0 2>$null; powercfg /setacvalueindex $guid 501a4d13-42af-4429-9fd1-a8218c268e20 ee12f906-d277-404b-b6da-e5fa1a55844a 0 2>$null; powercfg /setactive $guid; powercfg /list | Select-String "Power Scheme GUID" | Where-Object { $_ -notmatch $guid -and $_ -notmatch "8c5e7fda-e8bf-4a96-9a85-a6e23a8c635c" } | ForEach-Object { powercfg /delete ($_ -split '\s+')[3] 2>$null }; Write-Host "TARUS Power Plan Created, Activated & Enforced." -ForegroundColor Yellow

3. CNC Heartbeat Monitor (Ping Logger)

Stop via Ctrl+C

A standard Windows ping does not show when a network drop occurred. This script runs a continuous ping to the Column PC (192.168.214.55) and attaches a millisecond timestamp to every packet. Useful for proving to customer IT exactly when network drops are occurring.

ping 192.168.214.55 -t | ForEach-Object { if ($_ -match "Reply|Request|Destination") { "{0} - {1}" -f (Get-Date -Format "HH:mm:ss.fff"), $_ } }

4. Firewall Override (Netsh)

Temporary Override

Temporarily drops the Domain, Public, and Private firewalls to test blocked SProxy or Remote Desktop connections.

Duration Note: The firewall will remain disabled until the PC reboots, the customer's Group Policy (GPO) refreshes (usually ~90 mins), or you manually run the Restore Script below.

A. DISABLE FIREWALL:
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False; Write-Host "WARNING: All Firewalls Disabled." -ForegroundColor Red;
B. RESTORE FIREWALL:
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True; Write-Host "SUCCESS: Firewalls Restored to Secure State." -ForegroundColor Green;
